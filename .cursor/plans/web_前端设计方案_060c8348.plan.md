# HeartClaw Web 控制台 -- 前端设计方案

## 一、设计理念：「竹简工坊」

HeartClaw 的三个服务名字带有浓厚的东方匠人气质——如意（心想事成）、天工（巧夺天工）、锻造令（铸剑师的工单）。设计应该**延续这种工匠语境**，但作为开发者控制台，它需要在长时间使用中保持舒适。

核心思路：
- **深色暖调为主体**——不是冰冷的纯黑终端，而是带有墨色温度的深底
- **侧边栏是"脊柱"**——最深的色块，锚定整个界面
- **信息区域用"竹简纹理"**——微妙的行纹背景暗示，让平面不单调
- **强调色延续品牌**——赫石橙标记活跃操作，苔绿标记成功/在线
- **工具状态带微光**——深色底上的 badge 有轻微 glow effect，像锻造中的火星

## 二、色彩系统

### 主色盘（CSS Variables）

```css
/* -- 底色层 -- */
--bg-base: #13120f;          /* 最深底色，页面背景 */
--bg-surface: #1c1b17;       /* 面板/卡片底色，墨色偏暖 */
--bg-elevated: #252420;      /* 悬浮层/弹窗底色 */
--bg-sidebar: #0f0e0c;       /* 侧边栏，最深 */

/* -- 边框 -- */
--border-subtle: rgba(241, 234, 219, 0.06);   /* 分割线 */
--border-default: rgba(241, 234, 219, 0.12);  /* 卡片边框 */
--border-focus: rgba(163, 77, 36, 0.5);       /* 焦点态 */

/* -- 文字 -- */
--text-primary: #e8e0d0;     /* 主文字，暖白 */
--text-secondary: #8a8478;   /* 次要文字 */
--text-muted: #5c584f;       /* 最弱文字/placeholder */
--text-inverse: #13120f;     /* 深色文字（用于亮色按钮上） */

/* -- 强调色 -- */
--accent-orange: #c45a28;    /* 赫石橙，品牌主色/活跃态 */
--accent-orange-muted: rgba(196, 90, 40, 0.15);  /* 橙底色 */
--accent-green: #5d8a5e;     /* 苔绿，成功/在线 */
--accent-green-glow: rgba(93, 138, 94, 0.2);     /* 绿色微光 */
--accent-amber: #c49a3c;     /* 琥珀黄，警告 */
--accent-red: #b54a3a;       /* 窑红，错误 */
--accent-blue: #4a7a9b;      /* 青瓷蓝，信息/执行中 */

/* -- 工具状态专用色（带 glow） -- */
--status-validating: #c49a3c;
--status-awaiting: #c45a28;
--status-executing: #4a7a9b;
--status-success: #5d8a5e;
--status-error: #b54a3a;
--status-cancelled: #5c584f;
```

### 色彩使用原则

- 深色背景层级从深到浅：`sidebar -> base -> surface -> elevated`
- 文字永远是暖白色系，避免刺眼的纯白 `#fff`
- 强调色只用于**状态指示**和**可交互元素**，不大面积铺陈
- 赫石橙只出现在：当前选中的导航项、发送按钮、活跃状态 badge

## 三、字体方案

```css
/* 标题 / 导航 */
--font-display: "DM Serif Display", Georgia, serif;

/* 正文 / UI 元素 */
--font-body: "DM Sans", "Avenir Next", sans-serif;

/* 代码 / 日志 / 终端 */
--font-mono: "JetBrains Mono", "SF Mono", "Cascadia Code", monospace;
```

**选择理由：**
- **DM Serif Display** — 与现有的 Georgia 气质一脉相承，但更有设计感，笔画更有力量，适合作为品牌标题字体
- **DM Sans** — DM 家族的无衬线搭档，字形清晰干净，中等粗细时非常适合 UI 文字
- **JetBrains Mono** — 开发者最熟悉的等宽字体之一，连字 (ligature) 可选开启，代码和日志阅读体验极佳

Google Fonts 加载：
```html
<link href="https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500;700&family=DM+Serif+Display&family=JetBrains+Mono:wght@400;500;700&display=swap" rel="stylesheet">
```

```
+--------+

|--------|
|  如意   |   <-- MessageSquare 图标 + 文字
```

**侧边栏交互：**

## 五、如意聊天页

### 布局结构

```
+--------------------------------------------------+
|                   聊天消息流                        |
|                   (ScrollArea)                     |
|                                                    |
|  [用户] 你好，帮我创建一个 cron 任务                  |
|                                                    |
|  [如意] ▼ cron_create (executing)                  |  <-- 工具调用折叠块
|         参数: name="backup", cron="0 3 * * *"      |
|         ────────────────────                       |
|         好的，我已经为你创建了一个定时任务...            |
|                                                    |
+--------------------------------------------------+
| [输入框 _________________________ ] [发送]          |
+--------------------------------------------------+
```

### 消息气泡设计

**用户消息：**
- 右对齐
- 背景色 `--accent-orange-muted`（赫石橙的极淡底色）
- 圆角 16px，右下角 4px（指向用户方向）
- 文字色 `--text-primary`

**如意回复：**
- 左对齐
- 背景色 `--bg-surface`
- 内含 Markdown 渲染区域
- 回复中的代码块用 `--bg-base` 底色 + `--border-default` 边框

### 工具调用状态展示（紧凑型）

每次工具调用内联在如意的消息流中，以**可折叠块 (Collapsible)** 形式展示：

```
┌─────────────────────────────────────────────┐
│ ● bash                          executing ◉  │  <-- 折叠头：工具名 + 状态badge
├─────────────────────────────────────────────┤
│ $ ls -la /workspace                          │  <-- 展开后：参数摘要
│ ─────────────────────                        │
│ total 48                                     │  <-- 执行结果
│ drwxr-xr-x  6 root root 4096 ...            │
└─────────────────────────────────────────────┘
```

**状态 Badge 设计（带微光 glow）：**

| 状态 | 颜色 | 效果 |
|------|------|------|
| validating | 琥珀黄 | 脉冲动画 (pulse) |
| awaiting_approval | 赫石橙 | 闪烁 + 可点击"批准"按钮 |
| executing | 青瓷蓝 | 旋转的小圆点 (spinner) |
| success | 苔绿 | 静态勾号 + 淡绿色 glow |
| error | 窑红 | 静态叉号 + 淡红色 glow |
| cancelled | 灰色 | 静态横线 |

**Badge 的 CSS glow 效果示例：**
```css
.badge-executing {
  color: var(--accent-blue);
  background: rgba(74, 122, 155, 0.12);
  box-shadow: 0 0 8px rgba(74, 122, 155, 0.25);
}
```

### 输入区域

- 单行输入框，按 Enter 发送，Shift+Enter 换行
- 高度自适应（min 1 行，max 6 行）
- 底色 `--bg-surface`，焦点时边框变为 `--border-focus`
- 右侧发送按钮：赫石橙圆形按钮，内含箭头图标
- 发送中时按钮变为 spinner，禁用状态

## 六、天工日志页

### 布局结构

```
+--------------------------------------------------+
| [全部] [INFO] [WARN] [ERROR]   [⤓ 自动滚动] [暂停] |  <-- 过滤 tabs + 控制
+--------------------------------------------------+
|                                                    |
| 12:00:01.234  INFO   巡查开始，扫描 pending 目录...   |
| 12:00:01.456  INFO   发现锻造令: create_parser.md   |
| 12:00:02.100  INFO   移动到 processing/             |
| 12:00:05.789  WARN   Agent 输出超过 5000 字符        |
| 12:00:45.123  INFO   锻造完成，移动到 done/          |
| 12:01:00.000  ERROR  workspace 目录权限不足          |
|                                                    |
| █                                  <-- 光标闪烁     |
+--------------------------------------------------+
```

### 终端风格细节

**字体：** `JetBrains Mono`，14px，行高 1.6

**每行日志的结构：**
```
[时间戳]  [级别badge]  [消息内容]
```

**级别着色规则：**
- `INFO` — `--text-secondary`，最不显眼
- `WARN` — `--accent-amber`，琥珀黄文字
- `ERROR` — `--accent-red`，窑红文字 + 左侧 2px 红色竖条
- 时间戳 — `--text-muted`，最暗

**交互：**
- 自动滚动：新日志到达时自动滚到底部，用户手动上滚时暂停自动滚动，出现"回到底部"浮动按钮
- 日志级别过滤：顶部 tab 切换，当前选中的 tab 有赫石橙底色
- 搜索：可选的搜索框，高亮匹配文字
- 底部光标闪烁：最后一行显示一个闪烁的方块光标，暗示"实时连接中"

**背景纹理：**
日志区域的底色在 `--bg-base` 上叠加极淡的水平线纹（模拟终端扫描线），增加终端质感：
```css
background:
  repeating-linear-gradient(
    0deg,
    transparent,
    transparent 23px,
    rgba(241, 234, 219, 0.02) 23px,
    rgba(241, 234, 219, 0.02) 24px
  ),
  var(--bg-base);
```

### 文件树设计

- 背景色 `--bg-surface`
- 文件夹图标：lucide `FolderClosed` / `FolderOpen`
- 文件图标：根据扩展名变化（`.md` 用文档图标，`.yaml` 用齿轮图标）
- 当前选中文件：`--accent-orange-muted` 底色 + 左侧橙色竖条
- 缩进层级：每级 16px，用 `--border-subtle` 的竖线连接

### Monaco Editor 配置

- 主题：自定义暗色主题，匹配 HeartClaw 色盘
  - 编辑器背景 `--bg-base`
  - 选中行高亮 `--bg-elevated`
  - 注释色 `--text-muted`
  - 字符串色 `--accent-green`
  - 关键字色 `--accent-orange`
- 字体：`JetBrains Mono`
- minimap 关闭（节省空间）
- 显示行号

### 保存交互

- 文件修改后，底部状态栏显示橙色圆点 "已修改"
- 保存按钮：`Cmd+S` 快捷键绑定
- 保存成功：toast 提示 "已保存" + 苔绿色勾号
- 保存失败：toast 提示错误信息 + 窑红色叉号

## 八、动效方案

### 页面切换

- 路由切换时，内容区域做 **fade + 微距平移** 过渡：
  ```css
  /* 进入 */
  opacity: 0 -> 1, translateY(8px) -> 0
  duration: 200ms, ease-out
  ```
- 不使用夸张的滑动效果，保持工具属性的克制感

### 侧边栏

- 收起/展开：`width` 过渡 200ms ease
- 导航项 hover：背景色从 transparent 渐变到 `rgba(241, 234, 219, 0.06)`
- 选中指示条：赫石橙竖条 slide-in 动画

### 聊天

- 新消息到达：从底部 fade-in + translateY(12px)，duration 250ms
- 工具状态变更：badge 颜色过渡 150ms
- 输入框聚焦：边框 glow 渐现

### 日志

- 新日志行：从左侧微距 slide-in (translateX(-4px) -> 0)，快速 150ms
- 连接状态变化：底部状态点颜色过渡 + 一次脉冲动画

## 九、响应式策略

这是一个开发者控制台，主要在**桌面端**使用。移动端做基本适配但不是重点：

- **>= 1024px**：完整侧边栏 + 内容区域
- **768px - 1024px**：侧边栏默认收起为图标模式，点击展开为覆盖层
- **< 768px**：底部 tab 导航替代侧边栏，内容区域全屏
