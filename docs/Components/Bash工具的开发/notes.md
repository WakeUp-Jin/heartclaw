- `dangerouslyDisableSandbox` 用于控制是否启用沙箱执行，沙箱基于进程级限制实现：好处包括限制文件访问范围（默认只能读写工作目录及白名单路径，防止误删系统文件或读取敏感配置）、限制网络访问（防止恶意脚本外联）、限制高危系统调用（如 sudo、修改系统目录），且启动开销极低、无需像 Docker 那样拉取镜像。
- `_simulatedSedEdit` 是内部标记字段，用于将原本通过 Bash 执行的 `sed` 等文件编辑命令替换为更安全的内部编辑工具操作。
- `run_in_background` 设为 true 时，Bash 命令会在后台运行并返回任务 ID，后续通过 `TaskOutput` / `TaskList` / `TaskStop` 等任务工具查询或管理。【暂不实现，依赖底层任务模块支持】
